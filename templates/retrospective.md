# Template: Retrospective

## Suggested Topic

`What worked, what failed, and what should we change after <plenary/release/dogfood session>?`

## Framing Prompts

- What outcomes did we achieve?
- What friction cost us the most time?
- Which failures were process bugs vs product bugs?

## Divergence Prompts

- Name the top 1-3 changes with highest leverage.
- Which findings need GitHub issues vs docs updates vs protocol norms?
- What should we explicitly stop doing?

## Proposal Skeleton

`Retrospective outcome: keep <practices>, change <practices>, file issues for <bugs>, and update docs <files>. Next dogfood run will test whether <changes> reduced friction.`

## Resolution Checklist

- What worked
- What didn’t
- Concrete changes to process/tooling
- Issue/docs follow-ups captured
